# 核心技术

## 强一致

### 事务一致性
- 事务层采用优化的**二阶段提交协议**实现分布式事务
- 节点间的事务能根据数据分布方式自动进行事务切分
- 设置节点服务停止超时时间，不会让整个集群无限阻塞等待

### 数据一致性
- 数据层采用**数据流同步复制**方式结合**可靠 UDP 技术**
- 保证数据副本之间的强一致性

### 与主流数据库一致性对比

| 特性 | XuguDB | Oracle RAC | MySQL | PostgreSQL |
|------|--------|-----------|-------|------------|
| 分布式事务 | 优化二阶段提交 | 基于共享存储 | XA（性能差） | 2PC（需手动） |
| 数据副本一致性 | 同步复制 | 共享存储 | 半同步/异步 | 同步/异步 |
| 跨节点自动事务切分 | 支持 | 不需要（共享存储） | 不支持 | 不支持 |

## 高可用

### 数据存储高可用
- 采用数据存储**切片技术**实现分布式数据存储
- 灵活的数据版本管理，最多支持 **3 个数据存储版本**（1 主 + 2 备）
- 多版本数据存储模式下**主版本轮转分布，副本随机分布**
- 数据均匀分布到整个集群节点中

### 工作计算高可用
- 存算分离部署下，工作节点没有持久化数据
- 对外服务完全**对等同构**
- 用户连接到任意一个工作节点即连接到整个数据库集群
- 允许工作节点服务停止且不惧单点故障
- 所有工作节点服务停止也不会对数据存储造成影响

### 主控管理高可用
- 管理节点设计了**双活冗余**
- 当一个管理节点故障后，另一个自动接管

### 变更日志高可用
- 收集节点采用**主备双机**方案
- 保证事务粒度的同时保证可靠性

### 网络传输高可用
- 提供**多网络并行**功能
- 同一集群内部可配置多路网络进行通讯交互
- 为单网络故障提供容错
- 网络故障修复后可自动重新接入

## 高性能

### 事务技术

| 技术 | 说明 |
|------|------|
| **无限制级行级锁** | 管理锁粒度从对象级到行级，任何情况下不允许锁升级，大幅降低死锁可能性 |
| **基于代价的路径优化** | 根据数据分布、索引使用、分区情况、联合查询做最优路径选取 |
| **MVCC 多版本并发控制** | 维持数据的多个版本使读写操作无冲突，实现 Read Committed 隔离级别 |
| **集群可靠 UDP 技术** | 基于 UDP 封包格式，底层改造可靠 UDP 协议，大量减少性能损耗 |
| **RDMA 协议支持** | 直接内存访问，降低网络时延和 CPU 占用率；支持 InfiniBand 和 RoCE v2 |
| **存储数据压缩** | 数据去冗余压缩，加速数据传递效率 |
| **多核 NUMA 优化** | 关键数据结构分区，降低数据访问频次；关键业务线程绑核，避免核间线程漂移 |
| **元数据分布式管理** | 系统元数据表分散到各节点管理，避免元数据表成为性能瓶颈 |

### 分析技术

| 技术 | 说明 |
|------|------|
| **并行扫描** | 完全调度整个集群节点的计算和 I/O 能力处理 SQL 请求 |
| **智能单元扫描（算子下推）** | 基于"计算向存储靠拢"理念，数据在存储节点裁剪计算后再汇聚，减少网络开销 |
| **复杂统计分析函数** | 排名分析函数与聚合分析函数，保留所有原始行记录的同时进行分组数据分析 |

> **智能单元扫描**：当用户对数据表的部分字段与限制条件进行检索时，存储节点按指定输出列与限制条件进行数据裁剪，仅向工作节点返回少量符合需求的数据，以最少的资源消耗高效完成运算。

## 高安全

| 安全能力 | 说明 |
|----------|------|
| **自主研发安全可控** | 核心代码自有率 99.6%，未使用任何开源框架 |
| **复杂密钥身份验证** | 密码长度 [3,32]，支持字符/字母/数字混合；登录失败 3 次锁定 IP，3 分钟内禁用 |
| **自主访问控制（DAC）** | 基于用户权限的访问控制，权限记录在系统表（数据字典）中 |
| **标记强制访问控制（MAC）** | 利用策略和标记实现强制访问，控制粒度达到**记录级** |
| **存储传输加密** | 支持存储数据加密；支持 TCP/IP 和 SSL 会话 |
| **网络访问规则控制** | 黑白名单，对 Database 和 User 按 IP 地址或 IP 地址段进行安全管理 |

## 高扩展

- 基于无共享对等处理架构，拥有**动态增加节点**能力
- 计算能力不足 → 扩展工作节点
- 存储能力不足 → 扩展存储节点
- 可同时扩展计算和存储
- 计算节点不存储数据，扩展后即可参与计算任务
- 存储节点数据与逻辑表无关，按物理 CHUNK 存储，扩展后自动数据迁移均衡
- 数据采用 3 版本策略，存储误差比例控制在 **5% 以内**
- 支持**在线"热加入"**，无需停机
