# 虚谷数据库向量数据类型详解

XuguDB 提供 3 种向量数据类型：VECTOR、HALFVEC、SPARSEVEC，用于存储、管理和查询高维数值向量。

## VECTOR（32 位浮点稠密向量）

基于 32 位浮点数的稠密向量，每个维度均存储具体数值（即使为 0），是最通用的向量类型。

**格式**：使用 `[]` 表示，如 `[1,2,3,4]`

**核心特征**：
- 每个维度均包含实际数值（包括 0）
- 保留较高精度，适用于高精度计算或 ANN 检索
- 最大支持维度：16000

**创建方式**：
- `VECTOR(4)` — 指定维度为 4，插入数据的维度必须与之一致
- `VECTOR` — 不指定维度，可插入任意维度的向量

## HALFVEC（16 位半精度稠密向量）

基于 16 位半精度浮点数的稠密向量，在保留核心特征的前提下通过压缩数据格式优化存储。

**格式**：与 VECTOR 相同，使用 `[]` 表示，如 `[1,2,3,4]`

**核心特征**：
- 相比 VECTOR 节省约 50% 存储空间
- 更小的存储占用减少内存开销，检索吞吐更高
- 半精度浮点运算结果会自动取最接近的可表示值，可能与精确计算略有差异（正常行为）
- 最大支持维度：16000

**创建方式**：
- `HALFVEC(4)` — 指定维度为 4
- `HALFVEC` — 不指定维度

## SPARSEVEC（稀疏向量）

仅存储向量中非零值及其维度索引的向量类型，适用于"大部分维度为 0，仅少数维度有有效值"的稀疏数据。

**格式**：使用 `{}/` 表示，键为维度索引，值为该维度的数值，形式为 `index:value`

```
{1:1,3:2,6:5}/10
```
表示一个 10 维向量，索引位置 1、3、6 的值分别为 1、2、5，其余位置均为 0。

**核心特征**：
- 仅存储非零元素，非零值占比越低，存储优势越明显
- 索引不得重复
- 最大支持维度：10 亿

**创建方式**：
- `SPARSEVEC(4)` — 指定维度为 4
- `SPARSEVEC` — 不指定维度

## 类型对比

| 特性 | VECTOR | HALFVEC | SPARSEVEC |
|------|--------|---------|-----------|
| 精度 | 32 位浮点 | 16 位浮点 | 32 位浮点（非零值） |
| 存储方式 | 稠密 | 稠密 | 稀疏（仅非零值） |
| 最大维度 | 16000 | 16000 | 10 亿 |
| 存储效率 | 基准 | 约为 VECTOR 的 50% | 取决于稀疏度 |
| 适用场景 | 通用高精度检索 | 大规模数据、存储敏感 | 高维稀疏特征（NLP、TF-IDF） |

## DiskANN 索引维度限制

在创建 DiskANN 索引时，各类型的最大维度限制不同于数据类型本身的限制：

| 数据类型 | 索引最大维度 | 支持的距离度量 |
|----------|-------------|---------------|
| VECTOR | 2000 | L2、IP、Cosine、L1 |
| HALFVEC | 4000 | L2、IP、Cosine、L1 |
| SPARSEVEC | 1000（非零维） | L2、IP、Cosine、L1 |

## SQL 示例

### 创建向量表

```sql
-- 创建未指定维度的向量表
CREATE TABLE vec_tab_1(id INT, v1 VECTOR, v2 HALFVEC, v3 SPARSEVEC);

-- 创建指定维度的向量表
CREATE TABLE vec_tab_2(id INT, v1 VECTOR(2), v2 HALFVEC(3), v3 SPARSEVEC(4));
```

### 插入向量数据

```sql
INSERT INTO vec_tab_1 VALUES (1, '[1,2]', '[3,4,5]', '{1:6,3:7}/4');
INSERT INTO vec_tab_2 VALUES (1, '[1,2]', '[3,4,5]', '{1:6,3:7}/4');
```

### 查询向量数据

```sql
SELECT * FROM vec_tab_1;

+----+-------+---------+-------------+
| ID |  V1   |   V2    |     V3      |
+----+-------+---------+-------------+
| 1  | [1,2] | [3,4,5] | {1:6,3:7}/4 |
+----+-------+---------+-------------+
```

### 类型转换

使用 `::` 语法进行类型转换：

```sql
-- 字符串转向量类型
SELECT '[1, 2, 3]'::vector;
SELECT '[1, 2, 3]'::halfvec;
SELECT '{1:1, 3:2}/4'::sparsevec;
```

### 典型向量检索流程

```sql
-- 创建表
CREATE TABLE product_features (
    id INT PRIMARY KEY,
    product_name TEXT,
    feature_vector VECTOR(3)
);

-- 插入数据
INSERT INTO product_features (id, product_name, feature_vector) VALUES
    (1, '智能手机A', '[1.2, 3.4, 5.6]'),
    (2, '智能手机B', '[1.3, 3.5, 5.7]'),
    (3, '平板电脑A', '[7.8, 8.9, 9.0]'),
    (4, '笔记本电脑A', '[2.1, 4.3, 6.5]');

-- 检索最相似的前 2 个产品（L2 距离）
SELECT id, product_name, feature_vector <-> '[1.1, 3.3, 5.5]'::vector AS distance
FROM product_features
ORDER BY feature_vector <-> '[1.1, 3.3, 5.5]'::vector
LIMIT 2;

+----+--------------+--------------+
| ID | PRODUCT_NAME |   DISTANCE   |
+----+--------------+--------------+
|  1 | 智能手机A    | 1.732051e-01 |
|  2 | 智能手机B    | 3.464100e-01 |
+----+--------------+--------------+
```
