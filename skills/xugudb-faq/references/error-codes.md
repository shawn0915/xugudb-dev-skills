# 错误码参考

| 范围 | 分类 | 说明 |
|------|------|------|
| E1001-E1999 | 系统错误 | 内部错误、内存不足、资源限制 |
| E2001-E2999 | 连接/认证 | 连接超时、认证失败、会话错误 |
| E3001-E3999 | SQL 语法 | SQL 解析错误、语法不正确 |
| E4001-E4999 | SQL 语义 | 对象不存在、类型不匹配、表达式错误 |
| E5001-E5999 | DDL 操作 | 表/视图创建/删除错误 |
| E6001-E6999 | DML 操作 | 插入/更新/删除错误 |
| E7001-E7999 | 约束 | 主键/外键/检查约束错误 |
| E8001-E8999 | PL/SQL | 存储过程/函数/包编译运行错误 |
| E9001-E9999 | 对象管理 | 对象已存在、命名冲突 |
| E10001-E10999 | 权限 | 权限不足、访问被拒 |
| E11001-E11999 | 分布式 | 集群节点、分布式事务错误 |
| E12001-E12999 | 索引 | 索引创建/删除/重建错误 |
| E13001-E13999 | 唯一性 | 唯一索引/主键冲突 |
| E14001-E14999 | 锁 | 死锁、锁超时、锁冲突 |
| E15001-E15999 | 备份恢复 | 备份/恢复操作失败 |
| E16001-E16999 | 约束违反 | NOT NULL、CHECK 违反 |
| E17001-E17999 | 数据类型 | 类型转换、类型不存在 |
| E18001-E18999 | 权限详细 | 具体权限操作拒绝 |
| E19001-E19999 | 对象详细 | 触发器、同义词等特定错误 |
| E20001-E20999 | 事务 | 事务超时、事务状态错误 |
| E21001-E21999 | 空间数据 | GIS/空间功能错误 |
| E22001-E22999 | 向量 | 向量功能错误 |
| E24001-E24999 | 审计 | 审计策略/日志错误 |

## 常见错误码与解决方案

### DDL 错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E5021 | 表不存在 | 检查表名和 Schema；使用 `IF EXISTS` |
| E5025 | 表有依赖对象（外键/视图等） | 使用 `DROP TABLE t CASCADE` |
| E9016 | 对象已存在 | 使用 `IF NOT EXISTS`；先删除再创建 |

### 索引错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E12008 | 索引已存在 | 使用 `CREATE INDEX IF NOT EXISTS` |
| E13001 | 唯一性约束冲突 | 检查插入数据是否有重复值 |

### 约束错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E16005 | NOT NULL 约束违反 | 为 NOT NULL 列提供值 |
| E7xxx | 外键约束违反 | 确保引用的父表记录存在 |

### PL/SQL 错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E8007 | 包编译警告 | 检查包中的类型和引用 |
| E17002 | 类型不存在 | 检查数据类型名称拼写 |

### 权限错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E18012 | 权限不足 | 联系 DBA 授予所需权限 |
| E10xxx | 访问被拒 | 检查用户角色和权限设置 |

### 对象错误

| 错误码 | 含义 | 解决方案 |
|--------|------|----------|
| E19067 | 不支持的同义词目标 | 同义词不能指向触发器 |

## 错误处理最佳实践

### SQL 中忽略特定错误

```sql
-- 设置忽略特定错误码
SET EXCLUDE_ERRNO = 'E5021';  -- 忽略"表不存在"
DROP TABLE t;                  -- 不报错
SET EXCLUDE_ERRNO = '';        -- 清除设置
```

### 应用程序中捕获错误

```java
// Java JDBC
try {
    stmt.execute("DROP TABLE t");
} catch (SQLException e) {
    if (e.getErrorCode() == 5021) {
        // 表不存在，忽略
    } else {
        throw e;
    }
}
```

### PL/SQL 异常处理

```sql
BEGIN
    DROP TABLE t;
EXCEPTION
    WHEN OTHERS THEN
        DBMS_OUTPUT.PUT_LINE('错误: ' || SQLERRM);
END;
/
```

## 日志中查找错误

```bash
# 查找最近的错误
tail -100 /Server/XGLOG/ERROR.LOG

# 搜索特定错误码
grep "E13001" /Server/XGLOG/ERROR.LOG

# 按时间范围查找
grep "2026-04" /Server/XGLOG/ERROR.LOG | grep "E5"
```
