# 虚谷数据库 JDBC 常见问题与错误码

## 连接类错误

| 错误码 | 描述 | 原因 | 解决方案 |
|--------|------|------|---------|
| E50015 | URL 连接字符串中没有数据库 | 连接串缺少数据库信息 | URL 中必须包含数据库名称 |
| E50016 | URL 连接字符串中没有用户 | 连接串缺少用户名 | URL 中必须包含用户名 |
| E50017 | URL 连接字符串中没有密码 | 连接串缺少密码 | URL 中必须包含密码 |
| E50020 | 连接已关闭 | Connection 关闭后仍被使用 | 确认连接不再使用后再关闭 |
| E50025 | 无法连接上服务器 | IP/端口错误、服务器未启动、网络故障 | 确认网络正常、服务器已启动且可用 |
| E50026 | 无法获取服务器地址 | URL 中 host 信息错误 | 确保 URL 中 host 地址正确 |
| E50027 | 无法连接到指定服务器 | 用户名/密码/数据库名错误或权限不足 | 检查凭据和权限 |
| E50030 | host 信息中没有端口 | URL 格式错误 | 确保 URL 包含端口号 |
| E50073 | 无法加载驱动 | 找不到驱动类 | 检查 URL 是否正确、驱动包是否已导入 |
| E50074 | 连接信息、用户名、密码或为空 | 连接参数不完整 | 检查数据库是否存在、用户密码是否正确 |
| E50080 | 尚未和数据库建立连接 | 未成功建立连接 | 检查连接步骤是否正确 |

## 连接断开与重连

| 错误码 | 描述 | 原因 | 解决方案 |
|--------|------|------|---------|
| E50021 | 与服务器重新建立连接 | 网络闪断、服务器重启、连接超时 | 配置连接池活性检测；设置合理的空闲超时时间 |
| E50022 | 与服务器间的连接已断开 | 长时间无操作、网络断开、服务器关闭、超过 `connect_timeout` | 1) 修改 `max_idle_time`；2) 检查服务器状态；3) 调整 `connect_timeout` 参数 |
| E51023 | 重试 N 次无法连接上服务器 | 多 IP 集群中无可用节点 | 确保集群中 master 节点可用 |
| E51024 | 未给定连接 IP 信息 | 多 IP 配置格式错误 | 按手册书写正确的多 IP 连接信息 |

## 常见开发问题排查

### 1. 连接失败

```
E50025: 无法连接上服务器
```

排查步骤：
1. 确认服务器 IP 和端口是否正确（默认 5138）
2. 确认虚谷数据库服务是否已启动
3. 检查防火墙是否放行端口
4. 使用 `telnet serverIP 5138` 测试网络连通性
5. 检查驱动 JAR 包是否正确引入

### 2. 连接断开

```
E50022: 与服务器间的连接已经断开
```

排查步骤：
1. 检查数据库 `max_idle_time` 配置，适当增大
2. 使用连接池时，配置活性检测（`validateConAlive=true`）
3. 设置合理的 `connect_timeout`（`sockeTimeOut`）
4. 检查单次传入的参数个数是否超过 2048 个

### 3. 批量操作失败

排查步骤：
1. 检查是否已 `setAutoCommit(false)`
2. 检查批处理条数是否超过 32767
3. 确保异常时执行了 `rollback()`
4. PreparedStatement 批处理不支持 DDL 语句

### 4. 结果集更新失败

排查步骤：
1. 确认 URL 包含 `return_rowid=true`
2. 确认 Statement 创建时使用了 `ResultSet.CONCUR_UPDATABLE`
3. 确认查询为单表查询

### 5. 中文乱码

排查步骤：
1. 检查 URL 中 `charset` 参数（`utf8` 或 `gbk`，默认 `GBK`）
2. 确保客户端字符集与数据库字符集一致
3. Java 程序文件编码与数据库字符集匹配

### 6. 预处理语句参数错误

```
E50034: at least one parameter is not set by setXXX() method
```

排查步骤：
1. 检查 SQL 中 `?` 的个数与 `setXxx()` 调用次数是否一致
2. 确保所有参数都已设置后再调用 `execute()`
3. 参数下标从 1 开始，不是从 0 开始
